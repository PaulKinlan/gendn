Pointer event suppression on drag start

Chrome 147 fully implements the HTML specification requirement that, when a drag operation begins, the user agent must dispatch pointercancel, pointerout, and pointerleave events to the drag source. This signals that the pointer event stream has ended and no further pointer events will arrive for that pointer.

Behaviour change Code that handles drag-and-drop and also listens for pointercancel, pointerout, or pointerleave events may now receive those events at drag start in Chrome 147. This was previously partially implemented (mouse on non-Android and touch on Android) but is now applied consistently across all platforms.

why it exists

The HTML specification requires that when a drag-and-drop operation starts, the browser must tell the drag source that the pointer event stream is finished. This is done by dispatching pointercancel, pointerout, and pointerleave in sequence. Without this, the drag source is left in an ambiguous state — it still "thinks" the pointer is active and may hold state (pressed styles, hover highlights, internal gesture tracking) that never gets cleaned up.

Chrome had partially implemented this for mouse events on non-Android platforms and for touch drags on Android. Chrome 147 completes the implementation uniformly across all six supported Blink platforms.

what changes

Event sequence on drag start

When the user starts dragging an element, Chrome 147 now dispatches the following events to the drag source element, in order:

EventMeaning
pointercancelThe pointer event stream was cancelled (drag took over).
pointeroutThe pointer is no longer over the element.
pointerleaveThe pointer has left the element and all its ancestors.

Example: cleaning up pressed state

const handle = document.querySelector('.drag-handle');

handle.addEventListener('pointercancel', () => {
  // Fires at drag start in Chrome 147+ — clean up pressed/active UI state
  handle.classList.remove('is-pressed');
});

handle.addEventListener('pointerdown', () => {
  handle.classList.add('is-pressed');
});

// dragstart is dispatched after the pointer events above
handle.addEventListener('dragstart', (e) => {
  e.dataTransfer.setData('text/plain', handle.dataset.id);
});
