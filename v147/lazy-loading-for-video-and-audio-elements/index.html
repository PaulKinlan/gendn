Lazy loading for video and audio elements

Chrome 147 adds a loading="lazy" attribute to <video> and <audio> elements behind a flag, deferring media resource loading until the element nears the viewport. This matches the existing lazy-loading behaviour for <img> and <iframe>. The feature ships enabled by default in Chrome 148.

Developer trial — behind a flag In Chrome 147, loading="lazy" on <video> and <audio> is available behind a flag. Enable it at chrome://flags to test it locally. It ships by default in Chrome 148.
