Gamepad Event-Driven Input API

An extension to the Gamepad API that replaces polling (navigator.getGamepads()) with a rawgamepadinputchange event that fires synchronously on every hardware update — delivering lower-latency, lower-CPU gamepad input for games and assistive tools. Behind a flag in Chrome 147, origin trial in Chrome 149.

Developer trial — behind a flag The Gamepad Event-Driven Input API is in developer trial in Chrome 147. Enable the flag at chrome://flags to try it locally. The event name and interface shape may still change before general availability.

why it exists

The existing Gamepad API is polling-only: an application must call navigator.getGamepads() inside a requestAnimationFrame loop to read gamepad state. This introduces up to one frame of latency (16 ms at 60 Hz) and wastes CPU cycles even when no button is pressed. High-frequency controllers (e.g. 250 Hz gaming pads) can report 4 samples per frame that the polling model silently discards.

The event-driven model fires a rawgamepadinputchange event on window every time the hardware sends new data, regardless of the frame rate. Games and accessibility tools that need tight timing — fighting games, rhythm games, assistive devices — benefit immediately without rebuilding their entire input loop.

Source: chromestatus feature summary; web developer interest via Stack Overflow and w3c/gamepad#4.

example

// Requires: Gamepad Event-Driven Input flag enabled in Chrome 147
// Event-driven: fires at hardware report rate, not frame rate
window.addEventListener('rawgamepadinputchange', (event) => {
  const pad = event.gamepad;

  // Read axes (e.g. left stick: x, y)
  const [lx, ly] = pad.axes;
  moveCharacter(lx, ly);

  // Read buttons
  if (pad.buttons[0].pressed) {
    jump();
  }
});

// Polling still works and is unaffected by the new event
function gameLoop() {
  const pads = navigator.getGamepads();
  for (const pad of pads) {
    if (pad) renderHUD(pad);
  }
  requestAnimationFrame(gameLoop);
}
Pattern from W3C Gamepad spec; live demo at gabrielsanbrito.github.io/gamepad-raw-input-events
