PWA origin migration

A mechanism that lets an installed Progressive Web App seamlessly migrate to a new same-site origin — preserving user trust and permissions — without requiring the user to manually uninstall and reinstall. In developer trial in Chrome 147; ships by default in Chrome 150.

Developer trial — behind a flag PWA origin migration is in developer trial in Chrome 147. Enable the flag at chrome://flags to test the migration flow locally. The manifest field shape and browser UI may still evolve before Chrome 150, where it ships by default.

why it exists

An installed PWA's identity is tightly bound to its web origin — for example, app.example.com. When developers need to change origin (rebranding, domain restructuring, technical re-architecture), users are forced to uninstall the old app and reinstall from the new URL. This breaks local data continuity, home screen placement, and results in significant permanent user loss.

The PWA origin migration mechanism allows the developer to declare a migration target in the old app's Web App Manifest. Chrome detects the signal, verifies the new origin is same-site (same eTLD+1), and prompts the user to migrate with their permissions intact.

Enterprise constraint: Apps force-installed via the WebAppInstallForceList enterprise policy cannot be migrated. Chrome displays a banner explaining this to the user instead of offering the migration prompt.

example

The migration is declared in the old origin's Web App Manifest. The exact field name is still being finalised during the incubation period; refer to the WICG spec PR for the current shape.

// Old origin: app.example.com/manifest.json (conceptual shape)
{
  "name": "My App",
  "start_url": "/",
  "display": "standalone",

  // Declare that the app is moving to a new same-site origin.
  // Field name subject to change during incubation.
  "migration_target": "https://www.example.com/app/"
}

When Chrome detects this field (with the flag enabled in 147, or by default in 150), it shows the user a prompt to update the installed shortcut and origin. If the user accepts, the installed PWA continues from the new URL.

Conceptual example — refer to WICG manifest-incubations PR #136 and the sample app for authoritative field names.
