Web Neural Network API (WebNN)

A W3C API that lets web apps run neural network inference directly on a device's hardware — CPU, GPU, or dedicated NPU — through the browser. Frameworks like TensorFlow.js and ONNX Runtime Web use it under the hood; you can also use it directly to construct and execute computational graphs.

Origin trial The WebNN origin trial runs from Chrome 147 through 149. Register at developer.chrome.com/origintrials (search "WebNN") to get a token for production use. The API surface is stable within the current CR snapshot but still evolving — check the W3C spec before shipping. Enable locally: chrome://flags/#web-machine-learning-neural-network.

why it exists

Running ML models in the browser today requires either sending data to a server (latency, privacy) or using WebGL/WebGPU compute shaders (complex, not portable across frameworks). Native OS ML runtimes — DirectML on Windows, Core ML on macOS/iOS, NNAPI on Android — can run inference an order of magnitude faster than hand-written shader code, but the web has no standard way to reach them. WebNN is the bridge: it exposes a graph-based operator API that maps cleanly onto these native backends, letting frameworks compile an ONNX model once and run it optimally on whatever hardware the user has.

Source: W3C WebNN spec, webnn.io documentation, April 2026.

shape of the API

Core interfaces

InterfaceDescription
navigator.mlEntry point. Available in Window, Worker, and ServiceWorker contexts.
navigator.ml.createContext(options?)Creates an MLContext. options.deviceType can be "cpu", "gpu", or "npu".
MLContextRepresents the hardware backend. Has a compute() method to synchronously dispatch graph execution.
MLGraphBuilderConstructed with an MLContext. Provides methods for every operator (conv2d, relu, matmul, softmax, …). Returns MLOperand nodes forming a DAG.
MLGraphCompiled, immutable graph returned by MLGraphBuilder.build(). Execution is via MLContext.compute(graph, inputs, outputs).
MLTensorHardware buffer for sharing data between graph executions and other APIs (WebGPU, WebGL) without round-tripping through JS.
Source: W3C WebNN Candidate Recommendation, webnn.io API reference, April 2026.

example

Running a simple 2-layer network

// 1. Pick the best available device
const context = await navigator.ml.createContext({ deviceType: 'gpu' });
const builder = new MLGraphBuilder(context);

// 2. Describe the graph symbolically (values are shapes, not data)
const input  = builder.input('x', { dataType: 'float32', shape: [1, 784] });
const w1     = builder.constant({ dataType: 'float32', shape: [784, 128] }, weight1Data);
const b1     = builder.constant({ dataType: 'float32', shape: [128] }, bias1Data);
const fc1    = builder.add(builder.matmul(input, w1), b1);
const relu1  = builder.relu(fc1);
const w2     = builder.constant({ dataType: 'float32', shape: [128, 10] }, weight2Data);
const b2     = builder.constant({ dataType: 'float32', shape: [10] }, bias2Data);
const output = builder.add(builder.matmul(relu1, w2), b2);

// 3. Compile
const graph = await builder.build({ output });

// 4. Run
const inputTensor  = new Float32Array(784);
const outputTensor = new Float32Array(10);
// ... fill inputTensor with pixel values ...
await context.compute(graph,
  { x: inputTensor },
  { output: outputTensor }
);
console.log('Class probabilities:', outputTensor);
