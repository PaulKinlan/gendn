Clip Text overflow on user interaction

When a user clicks into or navigates by caret through an element showing a truncation ellipsis (text-overflow: ellipsis), Chrome temporarily switches from ellipsis to clip — letting the user see and interact with the full hidden content. In Chrome 147 this behaviour is available behind a flag; it ships by default in Chrome 149.

Developer trial — behind a flag This feature is in developer trial in Chrome 147. Enable the Clip text-overflow on user interaction flag at chrome://flags to try it. The API shape is expected to remain stable; the behaviour shipped by default in Chrome 149.

why it exists

text-overflow: ellipsis clips long text and replaces the tail with … — useful for labels and single-line display elements. The problem arises when the element is editable or keyboard-focusable: pressing End to jump to the end of a long filename, or clicking the middle of a truncated label in a WYSIWYG editor, moves the caret into invisible content. The user sees the cursor blink but cannot tell where they are.

The CSS Overflow spec has always required text to revert to clipping during user interaction, but the behaviour was only implemented for <input> and <textarea> form controls. This feature extends it to all elements — including contenteditable divs and any focusable element — matching what the spec requires.

Source: CSS Overflow Level 3 §ellipsis-interaction and chromestatus feature summary.

what changes

Before this feature Elements with text-overflow: ellipsis (other than <input> / <textarea>) kept showing … even when focused or edited, hiding the caret and part of the content.
With this feature enabled When the user focuses, clicks, or moves the caret inside such an element, the overflow mode temporarily switches to clip. The full text is visible during interaction. When focus leaves, the ellipsis is restored.
Affected elements All elements with overflow: hidden; white-space: nowrap; text-overflow: ellipsis that receive focus or editing — including contenteditable, elements with tabindex, and form controls (already worked before).
Source: CSS Overflow Level 3 §ellipsis-interaction.

example

In Chrome 147 with the flag enabled (or Chrome 149+ by default), clicking into the box below reveals the full text:

This is a very long line of text that gets truncated with an ellipsis in the display 
.label {
  overflow: hidden;
  white-space: nowrap;
  text-overflow: ellipsis;
  max-width: 260px;
  /* With flag enabled in Chrome 147 (or default in 149):
     clicking / tabbing into this element temporarily
     switches text-overflow to 'clip' so the caret is
     always visible and the full text is reachable. */
}

Contenteditable rich text field

<div
  contenteditable="true"
  style="
    overflow: hidden;
    white-space: nowrap;
    text-overflow: ellipsis;
    max-width: 300px;
  "
>
  A very long filename or title that might be truncated at rest
</div>

<!-- When the user clicks in to edit, the ellipsis disappears
     and the full text is visible. When focus leaves, it returns. -->
