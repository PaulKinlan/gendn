CSS URL request modifiers

The CSS url() function now accepts optional request modifiers — cross-origin(), integrity(), and referrer-policy() — that control how the browser fetches the referenced resource, bringing CSS resource loading to parity with HTML's <link> and <script> fetch attributes. Ships in Chrome 149 on all platforms.

why it exists

HTML has long allowed fine-grained fetch control on individual elements: <img crossorigin="anonymous">, <script integrity="sha384-...">, <link referrerpolicy="no-referrer">. CSS has no equivalent — a background-image: url("image.png") uses whatever defaults apply to the stylesheet's fetch context, with no way to request CORS mode, enforce subresource integrity, or suppress the referrer on a per-resource basis.

CSS URL request modifiers close this gap. The three modifiers map directly to the HTML attributes developers already know:

ModifierHTML equivalentControls
cross-origin()crossorigin="..."CORS mode for the fetch
integrity()integrity="..."Subresource Integrity hash check
referrer-policy()referrerpolicy="..."Referrer header sent with the request
Source: CSS Values 5 specification.

syntax

Modifiers are placed inside the url() function, after the quoted URL string, separated by whitespace:

url( <string> <url-modifier>* )

cross-origin()

/* CORS anonymous — no credentials sent */
background-image: url("https://cdn.example.com/hero.png" cross-origin(anonymous));

/* CORS with credentials */
background-image: url("https://cdn.example.com/profile.png" cross-origin(use-credentials));

Values match the crossorigin attribute: anonymous or use-credentials.

integrity()

/* Subresource Integrity check — resource must match the hash */
@font-face {
  font-family: "MyFont";
  src: url("https://cdn.example.com/font.woff2"
           integrity("sha384-oqVuAfXRKap7fdgcCY5uykM6+R9GqQ8K/uxy9rx7HNQlGYl1kPzQho1wx4JwY8wC"))
       format("woff2");
}

The hash format matches the integrity attribute on <script> and <link>.

referrer-policy()

/* Do not send any referrer header for this resource */
background-image: url("https://external.example/tracking-pixel.gif"
                       referrer-policy(no-referrer));

/* Use strict-origin for the referrer */
mask-image: url("mask.svg" referrer-policy(strict-origin));

Accepts the same values as the Referrer-Policy HTTP header and the HTML referrerpolicy attribute.

combining modifiers

background-image:
  url("https://cdn.example.com/image.png"
      cross-origin(anonymous)
      integrity("sha384-abc123...")
      referrer-policy(no-referrer));

which CSS properties can use modifiers

Any CSS property that accepts a url() value can use request modifiers, including:

  • background-image
  • mask-image, -webkit-mask-image
  • clip-path (referencing external SVG)
  • @font-face src
  • cursor
  • content
  • border-image-source, list-style-image
  • @import (for stylesheets)
