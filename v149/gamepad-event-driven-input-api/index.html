Gamepad Event-Driven Input API

A gamepadrawinputchanged event that fires whenever new gamepad input arrives — no polling loop required. Eliminates up to 8 ms of added latency for cloud gaming, competitive gaming, and any application that needs instant reaction to controller input.

Origin trial The Gamepad Event-Driven Input API is in origin trial in Chrome 149. Enable with flag chrome://flags/#gamepad-raw-input-change-event for testing. The API shape may change before GA. Check the explainer for the latest.

why it exists

The existing Gamepad API requires polling: you call navigator.getGamepads() in a requestAnimationFrame loop, typically at 60 Hz. The average latency between a physical button press and your code seeing it is half the polling interval — around 8 ms at 60 Hz. For cloud gaming, where network latency is already a concern, adding 8 ms of input delay in the browser is significant. Competitive games that poll at 120+ Hz to reduce this still burn CPU on every frame even when nothing has changed.

The event-driven API inverts the model: the browser calls your code exactly when the hardware reports new data. You get the lowest possible latency without wasting CPU on empty polls.

Source: GamepadEventDrivenInputAPI explainer §"Motivation".

shape of the API

Event: gamepadrawinputchanged

Fired on window whenever any connected gamepad reports new input data. The event delivers a snapshot of the gamepad state at the moment the hardware reported the change.

GamepadRawInputChangeEvent

gamepad Read-only Gamepad snapshot — the full state (axes, buttons, id, index, timestamp) at the moment of the change.
axesChanged number[] — indices of axes whose value changed in this event.
buttonsValueChanged number[] — indices of buttons whose value (analog pressure) changed.
buttonsPressed number[] — indices of buttons that transitioned to pressed in this event.
buttonsReleased number[] — indices of buttons that transitioned to released in this event.
touchesChanged number[] — indices of touch controls that changed (if the gamepad has a touchpad).
Source: GamepadEventDrivenInputAPI explainer §"Proposed solution".

example

Listen for any gamepad input change

window.addEventListener("gamepadrawinputchanged", (event) => {
  const pad = event.gamepad;

  // Handle axis changes (joystick movement)
  for (const axisIndex of event.axesChanged) {
    console.log(`Axis ${axisIndex}: ${pad.axes[axisIndex].toFixed(3)}`);
  }

  // Handle button presses
  for (const buttonIndex of event.buttonsPressed) {
    console.log(`Button ${buttonIndex} pressed`);
    handleAction(buttonIndex);
  }
  for (const buttonIndex of event.buttonsReleased) {
    console.log(`Button ${buttonIndex} released`);
  }
});

Cloud gaming — forward input immediately

window.addEventListener("gamepadrawinputchanged", (event) => {
  // Ship the full gamepad snapshot to the server immediately.
  // No wait for the next rAF frame.
  sendToServer({
    timestamp: event.gamepad.timestamp,
    axes: Array.from(event.gamepad.axes),
    buttons: event.gamepad.buttons.map(b => ({
      pressed: b.pressed,
      touched: b.touched,
      value: b.value,
    })),
  });
});

Mixed mode: event-driven + polling fallback

if ("ongamepadrawinputchanged" in window) {
  // Event-driven path (Chrome 149+ with origin trial)
  window.addEventListener("gamepadrawinputchanged", onInputChange);
} else {
  // Polling fallback for browsers without the event
  function pollLoop() {
    const pads = navigator.getGamepads();
    for (const pad of pads) {
      if (pad) onGamepadState(pad);
    }
    requestAnimationFrame(pollLoop);
  }
  requestAnimationFrame(pollLoop);
}
