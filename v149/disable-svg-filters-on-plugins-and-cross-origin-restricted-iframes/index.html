Disable SVG Filters on Cross-Origin Iframes and Plugins

Chrome 149 stops SVG filter effects defined on a parent page from visually altering cross-origin iframes and embedded plugins. The change closes a clickjacking vector where filter manipulation could disguise embedded content and trick users into unintended actions.

Possible breakage If your page applies CSS filter or SVG <filter> effects to a containing element and that element wraps a cross-origin <iframe>, the filter will no longer affect the iframe's rendered pixels in Chrome 149+. Audit layouts that overlay or style third-party embeds with filter effects.

why it exists

SVG filters can distort, recolor, and overlay arbitrary visual effects on any element, including the painted output of cross-origin iframes. An attacker's page can apply a CSS filter to a container that wraps a third-party login form, a payment button, or a CAPTCHA — making those elements look like something different while the clicks still go to the original content. This technique, called SVG clickjacking, lets attackers harvest credentials or trigger unintended authorizations.

Chrome 149 isolates cross-origin iframe and plugin rendering from the parent page's SVG filter graph. SVG filters stop at the boundary of a cross-origin frame: the iframe paints itself normally, and the filter is not composited over it. Safari already ships this behaviour.

Source: chromestatus feature 5117170452398080 and blink-dev Intent to Ship, April 2026.

what changes

Before Chrome 149 An SVG filter applied to an ancestor element painted over cross-origin iframes and plugins, visually altering their appearance.
Chrome 149+ SVG filters are stripped from the effect tree at the boundary of any cross-origin or sandboxed iframe and any embedded plugin (e.g. PDF). The frame renders without the filter.
Same-origin iframes Unaffected — filters continue to composite over same-origin content as before.

affected scenarios

ScenarioEffect in Chrome 149
Parent page uses filter: blur() on a wrapper div that contains a cross-origin <iframe> The iframe is not blurred. Only the same-origin content around it is blurred.
CSS backdrop-filter on a cross-origin iframe Not affected by this change — backdrop-filter is a separate mechanism.
Inline SVG <filter> applied via CSS filter: url(#myFilter) on a container The filter stops at cross-origin iframe boundaries.
PDF plugin embedded via <embed> or <object> SVG filters are stripped at the plugin boundary.
