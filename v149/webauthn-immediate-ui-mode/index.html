WebAuthn Immediate UI Mode

A new mediation: "immediate" value for navigator.credentials.get() that instantly surfaces saved passkeys or passwords if available — and silently rejects without showing a dialog if none are found. Zero friction for returning users; no blank modal for new ones.

why it exists

The existing mediation: "optional" flow shows a browser picker even when no credentials are stored. On a first visit that means users see a "sign in with passkey" dialog with nothing to sign in with — confusing and unnecessary. mediation: "conditional" (autofill) works without a modal but is tied to form input focus and doesn't suit button-triggered sign-in.

Immediate mode fills the gap: you call it when the user clicks a "Sign In" button. If Chrome has a passkey or saved password, it shows the credential sheet. If nothing is available, it rejects immediately with no UI — you fall through to your normal registration or password form. The result is a single sign-in path that is frictionless for returning users and invisible for first-time ones.

Source: WebAuthn immediate mediation explainer.

shape of the API

navigator.credentials.get() — mediation values compared

"silent" No UI, no user gesture needed. Returns credentials only if they can be provided without friction. Rarely useful for passkeys.
"optional" Shows a picker even when no credentials exist. Traditional flow.
"conditional" Passkeys/passwords appear as autofill suggestions in a <input autocomplete="webauthn"> field.
"immediate" ← new Shows the credential sheet if credentials are locally available; rejects with NotAllowedError if none found. Requires a user gesture. No dialog shown on rejection.

Promise resolution

Credentials found Chrome shows the credential picker. User selects a credential. Promise resolves with a PublicKeyCredential (passkey) or PasswordCredential.
No credentials found Promise rejects immediately with DOMException: NotAllowedError. No dialog is shown. The site handles this silently — typically by showing the standard sign-in form.
User cancels picker Promise rejects with NotAllowedError.

example

Button-triggered sign-in with passkeys + password fallback

signInButton.addEventListener("click", async () => {
  try {
    // Try immediate mode: shows picker only if credentials exist.
    const credential = await navigator.credentials.get({
      mediation: "immediate",
      publicKey: {
        challenge: await fetchChallenge(),
        rpId: location.hostname,
        userVerification: "preferred",
      },
    });
    // Credential obtained — verify on the server.
    await verifyCredential(credential);
    showDashboard();
  } catch (err) {
    if (err.name === "NotAllowedError") {
      // No credentials found: fall back to the normal sign-in form.
      showSignInForm();
    } else {
      throw err;
    }
  }
});

Unified passkey + saved-password flow

// Chrome 149 immediate mode works for both passkeys and saved passwords.
const credential = await navigator.credentials.get({
  mediation: "immediate",
  publicKey: {
    challenge: serverChallenge,
    rpId: location.hostname,
    userVerification: "preferred",
  },
  // Optional: also accept managed passwords
  password: true,
});

if (credential instanceof PasswordCredential) {
  // User picked a saved password.
  await loginWithPassword(credential.id, credential.password);
} else {
  // User picked a passkey — send assertion to server.
  await verifyPasskey(credential);
}

Combined with conditional UI for autofill

// On page load: set up conditional (autofill) mode so passkeys appear
// in the username input as the user focuses it.
navigator.credentials.get({
  mediation: "conditional",
  publicKey: { challenge, rpId: location.hostname, userVerification: "preferred" }
}).then(verifyAndRedirect).catch(() => {});

// On "Sign In" button: try immediate mode as a shortcut.
signInButton.addEventListener("click", async () => {
  try {
    const cred = await navigator.credentials.get({
      mediation: "immediate",
      publicKey: { challenge: await freshChallenge(), rpId: location.hostname }
    });
    await verifyAndRedirect(cred);
  } catch {
    // No immediate credentials — let the conditional flow handle it.
  }
});
