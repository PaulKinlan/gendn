Resource Timing: Service Worker Router Fields

Two new attributes on PerformanceResourceTiming — workerMatchedRouterSource and workerFinalRouterSource — expose which rule in the Service Worker Static Routing API matched a request and which source ultimately served it.

why it exists

The Service Worker Static Routing API lets a service worker declare URL-pattern-based rules at install time, bypassing the JS fetch event for matching requests. This can dramatically reduce resource load times by going straight to the network, cache, or fetch handler — without waking the service worker thread.

Before Chrome 149, there was no way to observe from outside the service worker which rule (if any) matched a request, or which source actually served the response. Developers had no visibility into whether their routing rules were firing correctly, making it hard to debug mismatches between intended and actual behaviour. The two new Resource Timing fields close that gap.

Source: chromestatus feature description and Chromium docs.

shape of the API

Both attributes are added to PerformanceResourceTiming (and by extension PerformanceNavigationTiming).

workerMatchedRouterSource string — identifies which router rule type matched the request. Values:
  • "network" — a rule directed the request to the network
  • "cache" — a rule directed the request to the Cache Storage
  • "fetch-event" — a rule directed the request to the service worker's fetch event handler
  • "race-network-and-fetch-handler" — a race rule was used
  • "" (empty) — no rule matched, or the service worker was not involved
workerFinalRouterSource string — identifies the source that ultimately served the response, after any fallbacks. Uses the same vocabulary as workerMatchedRouterSource. May differ if a rule's primary source failed and a fallback was used.
Source: W3C Resource Timing spec and chromestatus.

example

Observe router source for a specific request

const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    if (entry.entryType === "resource") {
      if (entry.workerMatchedRouterSource !== "") {
        console.log(
          `${entry.name}: matched="${entry.workerMatchedRouterSource}"` +
          ` final="${entry.workerFinalRouterSource}"`
        );
      }
    }
  }
});

observer.observe({ type: "resource", buffered: true });

Detect rule mismatches in production

// Log cases where the router went somewhere unexpected.
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    const { workerMatchedRouterSource: matched,
            workerFinalRouterSource:   final } = entry;
    if (matched && matched !== final) {
      // The matched source failed; browser used a fallback.
      analytics.log("sw_router_fallback", {
        url: entry.name,
        matched,
        final,
      });
    }
  }
});
observer.observe({ type: "resource", buffered: true });

Summarise router source distribution

const counts = { network: 0, cache: 0, "fetch-event": 0, other: 0 };
for (const entry of performance.getEntriesByType("resource")) {
  const src = entry.workerFinalRouterSource;
  if (src in counts) counts[src]++;
  else if (src) counts.other++;
}
console.table(counts);
