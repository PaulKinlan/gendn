Programmatic Scroll Promise

Programmatic scroll methods now return a Promise<undefined> that resolves when the scroll animation finishes — so you can sequence work after a smooth scroll without polling, guessing timing with setTimeout, or fighting the scrollend event.

why it exists

Before Chrome 149, all programmatic scroll methods returned undefined. There was no way to know when a smooth-scroll animation finished without approximating timing with setTimeout, watching the scrollend event (which fires for any scroll, not only programmatic ones), or using an IntersectionObserver. Chrome 149 makes every programmatic scroll method return a Promise that resolves when the animation completes.

Source: chromestatus feature 5082138340491264.

affected methods

MethodReturn type (Chrome 149+)

Element.scrollIntoView()Promise<undefined>
Window.scroll()Promise<undefined>
Window.scrollTo()Promise<undefined>
Window.scrollBy()Promise<undefined>

All methods resolve with undefined. The Promise resolves when the scroll position stops updating — whether the animation ran to completion, was interrupted by a user gesture, or required no movement at all (already at the target).

Source: CSSOM View Module scroll-promise-resolution algorithm.

example: await a smooth scroll before focusing

// Smooth-scroll an element into view, then focus it.
await element.scrollIntoView({ behavior: "smooth" });
element.focus(); // runs after scroll animation finishes

example: chain multiple programmatic scrolls

// Before Chrome 149: fire-and-forget, animations collide.
window.scrollTo({ top: 500, behavior: "smooth" });
window.scrollTo({ top: 1000, behavior: "smooth" }); // races the first

// Chrome 149+: await each step.
await window.scrollTo({ top: 500, behavior: "smooth" });
await window.scrollTo({ top: 1000, behavior: "smooth" });
console.log("reached bottom section");

example: lazy-load after scroll arrives

const section = document.querySelector("#lazy-section");
await section.scrollIntoView({ behavior: "smooth" });
// Fires exactly once the section is on screen.
await loadContent(section);

notes

  • Instant scrolls (behavior: "instant") return an already-resolved Promise; await continues in the next microtask.
  • If the element is already visible, scrollIntoView({ behavior: "smooth" }) still resolves immediately (no movement needed).
  • Interruption by a user gesture or a second programmatic scroll resolves the first Promise early — it does not reject.
