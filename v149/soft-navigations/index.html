Soft Navigations API

Brings Core Web Vitals measurement to Single Page Applications for the first time. A new SoftNavigationEntry PerformanceObserver type fires when an SPA transitions to a new "page", and a new interaction-contentful-paint entry measures its LCP-equivalent — without a full browser navigation.

Final origin trial The Soft Navigations API is in its final origin trial from Chrome 147 to 149, with stable ship expected after the trial concludes. Enable with chrome://flags/#enable-experimental-web-platform-features. Check the WICG repo for current status.

why it exists

Core Web Vitals — LCP, CLS, INP — are measured per browser navigation. For traditional multi-page apps, every link click is a navigation. But single-page apps (React, Vue, Angular, etc.) update the DOM and URL in JavaScript without triggering a browser navigation. From Chrome's perspective, the user is still on the original page. LCP is measured once (at load), and nothing captures the performance of subsequent "navigations" the user experiences.

The Soft Navigations API defines a heuristic: when a user interaction triggers a DOM change and a history.pushState/replaceState (or Navigation API update), Chrome emits a SoftNavigationEntry. This marks the start of a new "virtual page" for metric purposes. A subsequent interaction-contentful-paint entry measures the LCP of that virtual page. Developers and RUM tools can now measure SPA performance on the same level as MPA performance.

Source: WICG/soft-navigations explainer §"Motivation".

shape of the API

New PerformanceEntry types

soft-navigation Emitted when Chrome detects a soft navigation (user interaction + URL change + DOM change). Marks the start of a new virtual page session.
interaction-contentful-paint The LCP-equivalent for the virtual page — the largest paint after the interaction that triggered the soft navigation. One entry per soft navigation.

SoftNavigationEntry properties

name The new URL after the soft navigation (same as the new location.href).
startTime High-resolution timestamp of the user interaction that initiated the navigation.
duration Duration until the soft navigation was fully detected.
navigationId Unique identifier for this soft navigation. Cross-references to other entries (LCP, layout-shift) that belong to this navigation session.
interactionId The interaction ID (from Event Timing) of the user event that started the navigation.
Source: WICG/soft-navigations spec and Chrome implementation notes.

example

Observe soft navigations with PerformanceObserver

// Feature detection
if (PerformanceObserver.supportedEntryTypes.includes("soft-navigation")) {
  const observer = new PerformanceObserver((list) => {
    for (const entry of list.getEntries()) {
      console.log("Soft navigation:", {
        url: entry.name,
        startTime: entry.startTime,
        navigationId: entry.navigationId,
      });
    }
  });
  observer.observe({ type: "soft-navigation", buffered: true });
}

Measure LCP for each SPA route change

const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    if (entry.entryType === "soft-navigation") {
      console.log(`Navigation to ${entry.name} at ${entry.startTime}ms`);
    }
    if (entry.entryType === "interaction-contentful-paint") {
      console.log(`ICP for ${entry.url}: ${entry.startTime}ms`);
      sendToAnalytics("soft_nav_lcp", {
        url: entry.url,
        value: entry.startTime,
        navigationId: entry.navigationId,
      });
    }
  }
});

observer.observe({ type: "soft-navigation", buffered: true });
observer.observe({ type: "interaction-contentful-paint", buffered: true });

Slice CLS per soft navigation

// Track which soft navigation each layout shift belongs to.
const navsByTime = new Map();

new PerformanceObserver((list) => {
  for (const nav of list.getEntries()) {
    navsByTime.set(nav.startTime, nav.navigationId);
  }
}).observe({ type: "soft-navigation", buffered: true });

new PerformanceObserver((list) => {
  for (const shift of list.getEntries()) {
    // Find the navigation that "owns" this shift.
    const navId = [...navsByTime.entries()]
      .reverse()
      .find(([navTime]) => shift.startTime >= navTime)?.[1];
    if (navId) {
      accumulateCLS(navId, shift.value);
    }
  }
}).observe({ type: "layout-shift", buffered: true });
