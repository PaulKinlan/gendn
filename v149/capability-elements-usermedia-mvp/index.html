<usermedia> Element

A new HTML element that wraps the full lifecycle of a camera or microphone stream: one click requests permission and starts the stream; subsequent clicks mute and unmute. No more manual getUserMedia() + permission prompt + track management — just an element.

MVP — API surface may grow Chrome 149 ships the minimum viable <usermedia> element. The WICG proposal tracks further features (icon-only rendering, badge states, richer error feedback). Check the WICG spec before relying on anything beyond what is documented here.

why it exists

Requesting camera or microphone access today requires at least five separate steps: call getUserMedia(), handle the permission prompt, handle rejection, wire the resulting MediaStream into a <video> element, and implement separate mute/unmute logic. Each step has its own error paths and permission-state checks. The <usermedia> element collapses all of that into a single HTML element: the browser manages permission prompts, stream state, track lifecycle, and mute/unmute toggle. Sites that need straightforward "start cam / mute cam" UI can ship it with zero JavaScript for the core flow.

Source: WICG/PEPC usermedia_element.md §"Motivation".

shape of the API

HTML syntax

<usermedia>
  <script type="permissionconstraints">
    {
      "video": { "width": { "min": 640, "ideal": 1920 } },
      "audio": true
    }
  </script>
</usermedia>

The nested <script type="permissionconstraints"> carries the MediaStreamConstraints object. The element's type (video-only / audio-only / both) is inferred from the constraints.

Attributes

autostart Boolean. When present, the browser attempts to start the stream automatically when the element is added to the DOM (subject to permission state).

Properties

stream MediaStream | null — the active stream. Read-write: you can assign an externally-obtained MediaStream (e.g. after switching devices) without restarting the element.
enabled boolean — reflects the aggregate enabled/muted state of all tracks. Settable to mute or unmute programmatically.

Events

streamready Fired after the user grants permission and a MediaStream is successfully obtained. Access element.stream in the handler.
userchangedenabled Fired when the user clicks the element to toggle mute/unmute. Reflects the change in element.enabled.

Click behaviour

First click (no stream) Triggers getUserMedia() with the element's constraints. May show permission prompt. Sets element.stream and fires streamready on success.
Subsequent clicks (stream active) Toggles enabled on all associated MediaStreamTracks (mute / unmute). Fires userchangedenabled.
Source: WICG usermedia_element.md.

example

Video call: camera + mic with one element

<video id="preview" autoplay muted playsinline></video>

<usermedia id="my-camera" onstreamready="onStream()">
  <script type="permissionconstraints">
    { "video": true, "audio": true }
  </script>
</usermedia>

<script>
function onStream() {
  const el = document.getElementById("my-camera");
  if (el.error) {
    console.error("Permission denied:", el.error);
    return;
  }
  document.getElementById("preview").srcObject = el.stream;
}
</script>

Programmatic mute from separate button

const media = document.querySelector("usermedia");

document.querySelector("#mute-btn").addEventListener("click", () => {
  media.enabled = !media.enabled;
  this.textContent = media.enabled ? "Mute" : "Unmute";
});

media.addEventListener("userchangedenabled", (e) => {
  // Sync your UI if the user clicked the element itself.
  muteBtn.textContent = media.enabled ? "Mute" : "Unmute";
});

Device switching — reuse the element

const newStream = await navigator.mediaDevices.getUserMedia({
  video: { deviceId: { exact: selectedDeviceId } }
});

// Assign the new stream without recreating the element.
document.querySelector("usermedia").stream = newStream;

Auto-start on page load

<usermedia autostart>
  <script type="permissionconstraints">
    { "audio": true }
  </script>
</usermedia>
