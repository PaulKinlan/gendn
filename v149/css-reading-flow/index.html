CSS reading-flow and reading-order

Two CSS properties that decouple visual order from keyboard-navigation and screen-reader order in flex and grid containers. Set reading-flow on the container; optionally fine-tune individual items with reading-order.

why it exists

CSS flexbox and grid give authors powerful visual reordering tools — flex-direction: row-reverse, order, grid-template-areas — but these only affect painting. Keyboard focus and screen-reader order stay pinned to DOM order. This creates traps: a grid whose visual "first column" is structurally the last child, or a flex row-reverse that reads backwards via Tab. reading-flow lets the author explicitly declare which reading order the layout implies, and the browser will follow it for sequential navigation and assistive technology.

Source: CSS Display Level 4 §reading-flow motivation section.

the properties

reading-flow (container property)

Propertyreading-flow
Applies toBlock, flex, and grid containers
InheritedNo
Initialnormal
AnimationNot animatable
ValueEffect
normalFollow DOM order (default, existing behaviour).
source-orderCreates a reading flow container. Enables the reading-order property on direct children. Follows DOM order unless a child has an explicit reading-order.
flex-visualFlex containers only. Follows the visual order of flex items, taking writing mode into account. Tab order matches what you see on screen.
flex-flowFlex containers only. Follows the flex-flow direction.
grid-rowsGrid containers only. Follows visual order by row, taking writing mode into account.
grid-columnsGrid containers only. Follows visual order by column.
grid-orderGrid containers only. Follows order-modified document order (respects the order property, but not visual position).
Source: CSS Display Level 4 §propdef-reading-flow.

reading-order (item property)

Propertyreading-order
Applies toDirect children of a reading flow container (i.e. a container with reading-flow: source-order)
InheritedNo
Initial0
Value type<integer>
AnimationBy computed value

Sibling items are visited in ascending reading-order integer order. When two items share the same integer, the parent's reading-flow value breaks the tie. Mirrors the order property API but affects navigation order, not painting order.

Source: CSS Display Level 4 §propdef-reading-order.

example: row-reverse flex with matching tab order

Without reading-flow, a flex-direction: row-reverse row reads left-to-right visually but tabs right-to-left (DOM order). With reading-flow: flex-visual both visual and tab order move left-to-right.

.toolbar {
  display: flex;
  flex-direction: row-reverse; /* visually: D C B A */
  reading-flow: flex-visual;   /* tabs: D → C → B → A (left to right) */
}

/* Items in DOM order: A B C D */

example: grid layout with reading-order overrides

A sidebar-first DOM that renders content-first visually. reading-order ensures keyboard users reach the main content before the sidebar.

.layout {
  display: grid;
  grid-template-columns: 200px 1fr;
  reading-flow: source-order; /* enable reading-order on children */
}

.main-content {
  grid-column: 2;
  reading-order: 1; /* read first despite being second in DOM */
}

.sidebar {
  grid-column: 1;
  reading-order: 2; /* read second */
}
