HTML-in-Canvas

Draw live DOM elements — with full CSS layout and styling — directly into a 2D canvas, a WebGL texture, or a WebGPU texture. The rendered element stays interactive and accessible in the DOM; the canvas gets a pixel-perfect, GPU-composited snapshot.

Origin trial HTML-in-Canvas is in origin trial in Chrome 148–150. The API shape is still evolving. Check the WICG repo for the current spec before shipping. Flag: chrome://flags/#canvas-draw-element.

why it exists

Canvas-based UIs — rich text editors, game UIs, CAD tools, creative apps — have long needed to render HTML inside a canvas. The existing workaround is foreignObject inside an inline SVG serialised to a data URL and fed into drawImage(). That approach loses interactivity, breaks with cross-origin content, skips visited-link styling and accessibility, and can't be hardware-accelerated. HTML-in-Canvas gives apps a first-class path: children of a <canvas layoutsubtree> element lay out in the normal CSS cascade, remain part of the accessibility tree, and can be drawn into any canvas context at any time.

Source: WICG/html-in-canvas README "Motivation".

shape of the API

Step 1: opt the canvas into layout

layoutsubtree Boolean attribute on <canvas>. Direct children participate in CSS layout, hit testing, and the accessibility tree. Each child is a stacking context with paint containment.
canvas.requestPaint() Schedules a repaint. The paint event fires after intersection-observer steps complete.
onpaint / paint event Fires when any child element's rendering changes. event.changedElements lists which elements triggered the repaint.

Step 2: draw into the canvas context

2D context — drawElementImage

ctx.drawElementImage(element, dx, dy) Draw element at (dx, dy) at its natural size.
ctx.drawElementImage(element, dx, dy, dw, dh) Draw and scale to dw × dh.
ctx.drawElementImage(element, sx, sy, sw, sh, dx, dy, dw, dh) Source-crop then scale. Same parameters as drawImage() with an image source.
Returns DOMMatrix — the transform you can apply to element.style.transform to keep DOM position synchronised with the drawn position.

WebGL — texElementImage2D

gl.texElementImage2D(target, level, internalformat, format, type, element) Upload element's rendered pixels as a texture. Mirrors the texImage2D(target, level, iformat, format, type, image) signature.
With crop Accepts optional sx, sy, swidth, sheight before format, type, element to crop the source.

WebGPU — copyElementImageToTexture

device.queue.copyElementImageToTexture(source, destination) Copies source element pixels to destination (GPUImageCopyTextureTagged). Mirrors copyExternalImageToTexture().
With size Optional width, height and sx, sy, swidth, sheight overloads for cropping and resizing.

ElementImage — transferable snapshots

canvas.captureElementImage(element) Returns an ElementImage: a transferable snapshot of element's current render that can be postMessaged to a Worker for use with OffscreenCanvas.
canvas.getElementTransform(element, drawTransform) Returns the DOMMatrix mapping canvas draw coordinates to the element's DOM position.
Source: WICG/html-in-canvas WebIDL. Names may change before GA.

example

Render a form element into a 2D canvas on every paint

<canvas id="c" layoutsubtree width="600" height="400">
  <form id="form">
    <label>Name <input type="text"></label>
    <button>Submit</button>
  </form>
</canvas>

<script>
const canvas = document.getElementById("c");
const ctx = canvas.getContext("2d");
const form = document.getElementById("form");

canvas.onpaint = () => {
  ctx.reset();
  // Draw form at (20, 20); returns a matrix to keep DOM in sync.
  const transform = ctx.drawElementImage(form, 20, 20);
  form.style.transform = transform.toString();
};
</script>

Upload a DOM element as a WebGL texture

const gl = canvas.getContext("webgl2");
const texture = gl.createTexture();
gl.bindTexture(gl.TEXTURE_2D, texture);

canvas.onpaint = () => {
  // Upload the element as a texture on every repaint.
  gl.texElementImage2D(gl.TEXTURE_2D, 0, gl.RGBA, gl.RGBA, gl.UNSIGNED_BYTE, cardElement);
  renderScene();
};

Transfer a snapshot to an OffscreenCanvas worker

// Main thread
const snapshot = canvas.captureElementImage(richTextEditor);
worker.postMessage({ snapshot }, [snapshot]); // transferable

// Worker thread
self.onmessage = ({ data: { snapshot } }) => {
  const offscreen = new OffscreenCanvas(snapshot.width, snapshot.height);
  const ctx = offscreen.getContext("2d");
  ctx.drawElementImage(snapshot, 0, 0);
  // Process the pixels off the main thread.
};

security model

The API deliberately excludes several categories of sensitive information from drawn pixels to prevent side-channel leaks:

ExcludedCross-origin <iframe> / <img> content, visited-link styles, spelling markers, system colour values, subpixel anti-aliasing, pending autofill data.
IncludedSearch highlight / text-fragment markers, scrollbar appearance, caret blink phase, forced-colours media query state.
Source: WICG/html-in-canvas §5 "Security considerations".
