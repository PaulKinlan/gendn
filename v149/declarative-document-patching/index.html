Declarative Document Patching

Two complementary primitives for inserting HTML without heavy DOM manipulation: processing-instruction markers that let streaming HTML land out of order, and a revamped suite of JS methods (setHTML, streamHTML, appendHTML, …) with consistent naming, Sanitizer API integration, and Streams support.

Experimental Declarative Document Patching is available behind chrome://flags/#enable-experimental-web-platform-features from Chrome 148. The API names and shapes are still evolving in the WICG. Check the WICG repo for the latest before shipping.

why it exists

HTML is streamed top-to-bottom, but real applications often need to fill in a placeholder at the top of the page with data that arrives late — think server-streamed search results, AI-generated completions, or incremental database queries. Today that means buffering the full response, sending it as JSON, and imperatively updating the DOM with JavaScript. Declarative Document Patching provides two escape hatches: processing instructions let the server mark targets in the stream so that later chunks fill in earlier placeholders without JavaScript; the revamped dynamic markup API provides a richer, more secure replacement for innerHTML and insertAdjacentHTML with streaming, sanitization, and Trusted Types built in.

Source: patching-explainer.md §1 "Motivation".

shape of the API: processing instructions

Processing instructions are nodes in the HTML stream that mark named regions. Later <template for="name"> elements fill those regions — even if the template appears later in the document than the region it targets.

Processing instruction types

<?start name="id"?> Opens a named patchable range. Content between this and the matching <?end?> is the default (placeholder) content.
<?end?> Closes the nearest open <?start?> range.
<?marker name="id"?> Single insertion point. New content from a <template for> is inserted before the marker; the marker advances so sequential patches append in order.

<template for>

for Required. Names the <?start?> or <?marker?> target to patch.
contentrevision Optional. If the target already has this revision token, the patch is skipped (idempotent updates).
Source: patching-explainer.md §2–§4.

shape of the API: dynamic markup methods

All methods are on Element / ShadowRoot (the ElementOrShadowRoot mixin) and on ChildNode for positional insertion. Streaming variants return a WritableStream.

ElementOrShadowRoot — replace / append / prepend

setHTML(html, options?)Replace all children with sanitized HTML. Default sanitizer applied.
setHTMLUnsafe(html, options?)Replace children without mandatory sanitization. Accepts TrustedHTML.
appendHTML(html, options?)Append sanitized HTML.
appendHTMLUnsafe(html, options?)Append without sanitization.
prependHTML(html, options?)Prepend sanitized HTML.
prependHTMLUnsafe(html, options?)Prepend without sanitization.

ElementOrShadowRoot — streaming variants (return WritableStream)

streamHTML(options?)Replace children via a WritableStream. Progressive parse.
streamHTMLUnsafe(options?)Unsafe streaming replacement.
streamAppendHTML(options?)Append progressively via stream.
streamPrependHTML(options?)Prepend progressively via stream.

ChildNode — positional (before / after / replaceWith)

beforeHTML(html, options?)Insert before this node.
afterHTML(html, options?)Insert after this node.
replaceWithHTML(html, options?)Replace this node.
streamAfterHTML(options?)Stream HTML after this node; returns WritableStream.

SetHTMLOptions

sanitizerSanitizer | SanitizerConfig | "default" — defaults to "default" (built-in safe config).

example

Server-streamed out-of-order HTML (declarative)

<!-- Server sends this first: the page shell with a placeholder -->
<section>
  <?start name="results"?>
  <p>Loading…</p>
  <?end?>
</section>

<!-- …other content loads here… -->

<!-- Server streams this later, targeting the placeholder above -->
<template for="results">
  <ul>
    <li>First result</li>
    <li>Second result</li>
  </ul>
</template>

Appending a live feed with streamAppendHTML

const feed = document.querySelector("#feed");
const writer = feed.streamAppendHTML().getWriter();

// Pipe a fetch response directly into the DOM.
const response = await fetch("/api/live-items");
const reader = response.body.pipeThrough(new TextDecoderStream()).getReader();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  await writer.write(value);
}
await writer.close();

Safe innerHTML replacement with setHTML

// Replaces innerHTML with default sanitization — no XSS from user content.
container.setHTML(userProvidedMarkup);

// Custom sanitizer config.
container.setHTML(userMarkup, {
  sanitizer: { allowElements: ["p", "b", "i", "a"] }
});
