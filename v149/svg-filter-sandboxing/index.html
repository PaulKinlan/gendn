SVG Filter Sandboxing

Chrome 149 stops SVG filter effects from being applied across origin boundaries. Filters defined on a parent page can no longer bleed into cross-origin <iframe> elements or embedded plugins (PDFs). Safari already ships this isolation; Chrome now aligns.

Breaking change If your app wraps cross-origin iframes with an SVG filter (blur, colour matrix, etc.), those filters will silently stop applying in Chrome 149. Check enterprise dashboards for affected users — Google rates the enterprise impact as High.

why it exists

CSS filter and SVG filter effects traverse the compositing tree when rendering. Before Chrome 149, a parent-page filter could apply to the painted pixels of a cross-origin child frame — leaking that the frame had rendered at all (useful for timing attacks) and allowing a parent to visually manipulate another origin's content (potentially misleading users). Safari already scopes SVG filters to their origin boundary. Chrome 149 closes the gap.

The specific mechanism: when building the effect tree, Chrome now detects that a filter would cross an origin boundary or reach a sandboxed frame/plugin, and it skips the filter for those subtrees. The filter is effectively a no-op for cross-origin content; same-origin content is unaffected.

Source: Chrome 149 beta release notes and enterprise impact assessment.

what changes

Before Chrome 149 A parent-frame element with filter: blur(4px) or an SVG filter applied to it could visually blur a cross-origin child <iframe> or embedded PDF plugin rendered within it.
Chrome 149+ When a filter effect would cross an origin boundary (cross-origin iframe or plugin), Chrome finds the highest ancestor without SVG filters and applies the effect only up to that point. Cross-origin frames and plugins are rendered without the filter.
Same-origin iframes Unaffected — filters still apply normally within the same origin.

affected patterns

Blurring a cross-origin embed Stops working. E.g., blurring a payment iframe or third-party widget.
CSS filter on a container wrapping mixed-origin content Filter silently removed from cross-origin descendants.
SVG <feBlend>, <feColorMatrix>, etc. over cross-origin content Stops working.
Filters applied to same-origin iframes Unaffected.
Filters applied to elements with no cross-origin children Unaffected.

Mitigation

/* Before — tried to blur a cross-origin iframe */
.embed-wrapper {
  filter: blur(4px) grayscale(0.5);
}
.embed-wrapper iframe { /* cross-origin */ }

/* After — move the filter to a separate overlay instead */
.embed-wrapper {
  position: relative;
}
.embed-wrapper::after {
  content: "";
  position: absolute;
  inset: 0;
  backdrop-filter: blur(4px) grayscale(0.5);
  pointer-events: none;
}

/* Note: backdrop-filter blurs what is visually "behind" the pseudo-element,
   which may or may not behave the same as filtering the iframe directly.
   For a true blur overlay, a canvas overlay may be needed. */
