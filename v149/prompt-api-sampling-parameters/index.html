Prompt API Sampling Parameters

An extension to the Prompt API (LanguageModel) that exposes temperature and topK sampling parameters, letting sites control how creatively or predictably the on-device model responds — and query the model's defaults and supported ranges.

Origin trial Prompt API Sampling Parameters are in origin trial (Chrome 148–153). Sampling parameters were deliberately excluded from the initial Prompt API ship due to interoperability concerns across models and browsers. The OT gathers data before standardisation. Check the chromestatus entry for current status.

why it exists

Language models use sampling algorithms to pick the next token. Two key parameters govern this:

  • temperature: scales the probability distribution over candidate tokens. A lower value (e.g. 0.2) makes the model more deterministic and focused; a higher value (e.g. 1.2) makes it more varied and creative.
  • topK: limits sampling to the K most likely tokens at each step. A lower K (e.g. 5) increases coherence; a higher K allows more diversity.

The initial Prompt API ship intentionally omitted these parameters because their semantics — and safe defaults — vary by model and implementation, making cross-browser standardisation difficult. The origin trial exists to gather production data on how developers use sampling parameters before the values and ranges are locked into the spec.

shape of the API

Creating a session with custom sampling parameters

const session = await LanguageModel.create({
  temperature: 0.8,  // float: 0.0 = deterministic, higher = more creative
  topK: 10           // integer: number of top token candidates to sample from
});

// Parameters are readable on the session
console.log(session.temperature); // 0.8
console.log(session.topK);        // 10

Querying model defaults and limits

const params = await LanguageModel.params();

console.log(params.defaultTemperature); // e.g. 1.0
console.log(params.maxTemperature);     // e.g. 2.0
console.log(params.defaultTopK);        // e.g. 3
console.log(params.maxTopK);            // e.g. 8

Full example

async function runWithSampling() {
  const params = await LanguageModel.params();

  // Creative mode: higher temperature, wider topK
  const creativeSession = await LanguageModel.create({
    temperature: Math.min(1.5, params.maxTemperature),
    topK: Math.min(20, params.maxTopK)
  });

  const story = await creativeSession.prompt(
    'Continue this story: The lighthouse keeper noticed something strange...'
  );

  creativeSession.destroy();
  return story;
}
