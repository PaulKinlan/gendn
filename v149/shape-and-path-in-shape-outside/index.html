shape() and path() in shape-outside

Chrome 149 lets text flow around arbitrary curves, arcs, and bezier paths using the shape() and path() functions as values for shape-outside. The same functions already worked in clip-path since Chrome 137 — now the layout engine wraps text around them too.

why it exists

CSS Shapes Level 1 (shipped in Chrome 37) lets text flow around floated elements with circle(), ellipse(), polygon(), and inset() shapes. That covers rectangles with rounded corners and simple symmetric shapes, but not organic curves — the kind used in editorial layouts where text wraps around a complex illustration. The shape() function (CSS Shapes Level 2) and path() (an SVG-borrowed function) were already available in clip-path and offset-path. Chrome 149 extends both to shape-outside, completing the picture: whatever you can clip to, you can now wrap text around.

Source: CSS Shapes Module Level 2 introduction and blink-dev Intent to Ship.

shape() function syntax

shape() is a declarative alternative to path(). It uses CSS coordinates with units, percentages, and calc(); path() uses raw SVG coordinates. shape() is generally preferred for new code.

from <point> Required first command. Sets the starting point of the shape.
line to <point> Straight line to the given point.
curve to <point> via <cp1> Quadratic Bézier to <point> with control point <cp1>.
curve to <point> via <cp1> <cp2> Cubic Bézier with two control points.
smooth to <point> Smooth cubic Bézier that mirrors the previous control point (like SVG S).
arc to <point> of <rx> [<ry>] Elliptical arc. Optional flags: cw / ccw (sweep), large / small (arc size).
close Closes the path back to the starting point.
Source: CSS Shapes Level 2 §shape-function.

path() syntax

path() accepts an SVG path data string. Coordinates are in user units (pixels at 1× zoom), not percentages. Works best when the shape is defined in SVG tooling.

path("M 0 80 Q 80 0 160 80 L 160 160 L 0 160 Z") Quadratic arc from SVG path data. Same syntax as the SVG d attribute.

Also added in Chrome 149: rect() and xywh()

rect(top right bottom left)Inset rectangle with round corner support — a named alternative to inset().
xywh(x y w h)Rectangle by x/y offset and dimensions instead of insets.
Source: CSS Shapes Level 2.

example

Text flowing around a curved float

Text wraps around the curved underside of this shape. The float box is 160 × 160 px; the shape-outside defines an arch that opens upward, so text runs under the arch and fills the corners. No image, no mask — just a shape() value on a plain div.

.float-shape {
  float: left;
  width: 160px;
  height: 160px;
  margin: 0 1rem 0.5rem 0;

  /* Arch opening upward: text fills below and in the corners */
  shape-outside: shape(
    from 0% 100%,
    curve to 100% 100% via 50% -30%,
    line to 0% 100%
  );
}

Organic text wrap with path()

/* A wave shape defined with SVG path data */
.wave-float {
  float: left;
  width: 200px;
  height: 150px;
  shape-outside: path("M 0 150 C 50 80, 150 80, 200 150 Z");
  shape-margin: 8px;
}

Magazine-style layout

.hero-image {
  float: left;
  width: 300px;
  height: 400px;
  shape-outside: shape(
    from 0% 0%,
    line to 60% 0%,
    arc to 60% 100% of 80px large cw,
    line to 0% 100%,
    close
  );
  shape-margin: 12px;
}
