Web App Origin Migration

Lets a Progressive Web App move to a new origin without forcing users to uninstall and reinstall. The old and new manifests declare a two-way relationship; the browser validates them, then offers or performs the migration, preserving the app's identity, trust, and permissions on the user's device.

why it exists

A PWA's identity is bound to its origin. Rebranding (changing domain) previously forced users through a manual uninstall-and-reinstall cycle, causing app data loss and drop-off. Web App Origin Migration introduces a manifest-declared, .well-known-validated handshake so the browser can migrate an installed app to a new origin on the user's behalf — with appropriate consent UI for cross-origin moves.

Source: WICG manifest-incubations explainer, May 2026.

how it works

Migration requires a two-way handshake: both the new app's manifest and the old origin's .well-known file must declare the relationship, preventing any third party from silently hijacking an installed app's identity.

1. New app manifest — migrate_from

{
  "name": "My App",
  "id": "/app/",
  "start_url": "/app/",
  "migrate_from": [
    {
      "id": "https://old.example.com/app/",
      "behavior": "suggest"
    }
  ]
}

2. Old origin — /.well-known/web-app-origin-association

{
  "web_apps": [
    {
      "manifest": "https://new.example.com/app/manifest.json",
      "details": {
        "allow_migration": true
      }
    }
  ]
}

3. Old app manifest — migrate_to (optional)

The old app can proactively trigger migration discovery during its next manifest update check:

{
  "name": "My App (legacy)",
  "id": "/app/",
  "migrate_to": {
    "id": "https://new.example.com/app/",
    "install_url": "https://new.example.com/app/"
  }
}

manifest fields

FieldInDescription
migrate_fromNew manifestArray of old-origin app identifiers this new app succeeds. Requires id of the old app and optional behavior.
migrate_toOld manifest (optional)Proactively points to the new app; triggers migration check on next update.
behaviormigrate_from entry"suggest" — passive notification the user can ignore. "force" — blocking dialog on next app launch.
idBoth manifestsRequired in both old and new manifests for migration to function.

browser discovery flow

PathTriggerProcess
Reactive User visits new site Browser fetches new manifest, sees migrate_from, validates against old origin's /.well-known/web-app-origin-association, then presents migration UI.
Proactive Old app's periodic manifest update check Browser detects migrate_to in old manifest, verifies reciprocal migrate_from in new manifest, initiates migration flow.
