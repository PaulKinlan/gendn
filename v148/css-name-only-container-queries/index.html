CSS Name-only Container Queries

A container can now be targeted by container-name alone in a @container rule, without requiring a container-type to be set. This relaxes the constraint that a size query must have a containment context, enabling name-based scoping of style queries and other non-size query types.

this feature is documented on MDN

MDN: @container

MDN's @container page covers the full container query syntax including name-only queries, style queries, and the container-name / container-type properties. gendn doesn't duplicate that here.

quick reference

Before (Chrome 147)Both container-name: sidebar AND container-type: inline-size were required to use @container sidebar (...)
After (Chrome 148)container-name: sidebar alone is enough — no container-type needed for name-only queries
Applies toStyle queries and any query that does not need dimensional containment
Example.sidebar { container-name: sidebar; }
@container sidebar style(--variant: compact) { … }
