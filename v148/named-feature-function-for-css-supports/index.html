named-feature() function for CSS @supports

New functions added to CSS @supports for feature-detecting at-rules and named CSS features — at-rule() tests whether the browser supports a specific CSS at-rule (like @layer or @container), and named-feature() tests support for named features. Shipped to Chrome 148.

this feature is documented on MDN

MDN: @supports

MDN's @supports page covers all feature test functions including selector(), at-rule(), and named-feature(), with browser compatibility and practical examples. gendn doesn't duplicate that here.

quick reference

at-rule(<at-keyword>)Returns true if the browser supports the CSS at-rule with that keyword. Example: @supports at-rule(@layer) { … }
named-feature(<name>)Returns true if the browser recognises the named CSS feature. Example: @supports named-feature(view-transitions) { … }
