Opaque origin for data: URL Dedicated and Shared Workers

Chrome 150 aligns with the HTML specification and other browsers by giving Dedicated and Shared Workers created from data: URLs a unique opaque origin instead of inheriting the creator's origin — a security fix that breaks sites relying on same-origin access from data:-URL workers.

the change

Before Chrome 150, a Dedicated or Shared Worker created with a data: URL inherited the security origin of the JavaScript context that created it:

// Old Chrome behaviour (non-standard):
// This worker's origin === window.location.origin
const worker = new Worker("data:text/javascript,self.postMessage(location.href)");
// or
const shared = new SharedWorker("data:text/javascript,...");

This non-standard behaviour allowed data:-URL workers to access same-origin resources — BroadcastChannel, localStorage, IndexedDB — as if they were trusted scripts on the host page.

Chrome 150 fixes this: a worker created from a data: URL now receives a unique opaque origin (the same as null for document.domain purposes). This matches Firefox, Safari, and the HTML specification.

Source: blink-dev Intent to Ship.
breaking change Code that creates a data:-URL worker and then uses BroadcastChannel, localStorage, IndexedDB, or any other same-origin-gated API from inside the worker will break in Chrome 150. The worker's origin is now opaque and those calls will be blocked.

what breaks

PatternEffect after Chrome 150
new BroadcastChannel("ch") inside a data: workerBlocked — opaque origins cannot join named broadcast channels

fix: use a blob: URL instead

If you need the worker to share the creator's origin, create it from a blob: URL instead of a data: URL. A blob: URL inherits the origin of its creating context and preserves same-origin access:

const workerCode = `
  const channel = new BroadcastChannel("my-channel");
  channel.postMessage("ready");
`;

// data: URL (breaks in Chrome 150 — opaque origin)
// const worker = new Worker("data:text/javascript," + workerCode);

// blob: URL (correct — inherits creator's origin)
const blob = new Blob([workerCode], { type: "text/javascript" });
const url = URL.createObjectURL(blob);
const worker = new Worker(url);
// Remember to revoke when done:
URL.revokeObjectURL(url);
